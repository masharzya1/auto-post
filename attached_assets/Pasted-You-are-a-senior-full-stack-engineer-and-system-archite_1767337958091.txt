You are a senior full-stack engineer and system architect.

Build a production-grade web application using:
- React (Vite)
- Plain JavaScript only (NO TypeScript)
- Node.js
- Firebase (Firestore)
- Deployable on Vercel
- Client-side heavy processing (no long server rendering)

--------------------------------
PROJECT OVERVIEW
--------------------------------
This app is an AI-powered content automation platform inspired by n8n,
designed for NON-TECHNICAL users.

The system allows a single owner to:
- Configure all automation settings once
- Store configuration in Firebase
- Run cron-based automation even when browser is closed
- Generate AI text, images, and SHORT videos
- Generate videos directly inside the browser
- Prepare content for Facebook and YouTube Shorts

--------------------------------
DATA STORAGE RULES
--------------------------------
- Firebase Firestore MUST be used
- No other database allowed
- Firestore is the SINGLE SOURCE OF TRUTH
- localStorage may be used only for UI cache

--------------------------------
FIREBASE DATA MODEL (MANDATORY)
--------------------------------

Collection: settings
Document: main

Fields:
- fbPageId (string)
- ytChannelId (string)
- niche (string)
- postsPerWeek (number)
- videosPerDay (number)
- randomPostingTime (boolean)

Collection: limits
Document: daily

Fields:
- textLimit (number)
- imageLimit (number)
- videoLimit (number)
- textUsed (number)
- imageUsed (number)
- videoUsed (number)
- lastResetDate (timestamp)

Collection: workflows
Document per workflow

Fields:
- name (string)
- enabled (boolean)
- lastRun (timestamp)

--------------------------------
CRON JOB SYSTEM (VERY IMPORTANT)
--------------------------------

Implement cron jobs using:
- Vercel Cron Jobs (preferred)
OR
- node-cron (for Replit development)

Cron job responsibilities:
- Run every 15 or 30 minutes
- Read schedule & limits from Firestore
- Decide:
  - Should content be prepared now?
  - Are daily limits exceeded?
- Update Firestore with:
  - textUsed
  - imageUsed
  - videoUsed
  - lastRun timestamps

Include:
- Cron setup code
- Clear comments explaining logic
- Explanation of which Firestore fields are read/written

--------------------------------
VIDEO GENERATION (MANDATORY)
--------------------------------
- Video generation MUST happen inside the browser
- Use ffmpeg.wasm
- No server-side video rendering

Video requirements:
- Vertical format (9:16)
- 15–30 seconds
- Images + text overlays
- Bangla subtitles burned
- Fade transitions
- Zoom / pan effect
- Progress bar while rendering

User flow:
- Dashboard shows "Video Ready"
- User clicks "Generate Video"
- Browser renders MP4
- File becomes downloadable / upload-ready
